Swap Your Arrays For Sets

Committed 8. July 2019. 6 min parse.

Swap All The Arrays!

Well no, chances are you won't need to convert all of your existing Arrays to Sets, you might not even need to any but you should definitely see if the points below apply to you. Let's first discuss what a Set is and how it differs from an Array. A Set, like an Array, is a collection of any type of values that can be iterated through. The biggest difference between Sets and Arrays is that a value in a Set can only be defined once and what this means is that every value in a Set is unique. On a lower level, you can think of this "every value in a Set is unique" in terms of a strict equals being used when an item is inserted into a Set. For example, if you tried to add the number 5 to a Set twice, it would add it the first time and then just discard it the second time because it is not a unique value within the Set anymore. This is not a perfect system though and it won't always work with all value types so make sure to check the extra information at the bottom for more details because it's time we get to some examples!

Using Sets

So to create a Set you have to create a new instance of one like so:

const numbers = new Set();

You can also initialize it with a set of values wrapped in an Array from the start like so:

const numbers = new Set([1, 2, 3, 4, 5]);

And then from here on out, the basic API is very simple.

Adding Values

To add a new value to the set, you just have to call the add method like so:

numbers.add(6);

// Set [1, 2, 3, 4, 5, 6]

To emphasize the no duplicate value rule, doing the following:

numbers.add(7);
numbers.add(7);

// Set [1, 2, 3, 4, 5, 7]

will result in the first 7 being added to the Set and the second 7 will be just silently ignored.

Checking If A Value Exists

If you want to check if a specific value exists, you can use the has method like so:

const hasFive = numbers.has(5);

// true

and it doesn't have to be a direct value, you can use a variable from a loop or anywhere else as the argument.

Deleting A Value

To delete a value from the set, you just have to call the delete method with the value you want to delete as the argument:

numbers.delete(7);

// Set [1, 2, 3, 4, 5, 6]

Note that if you know the exact value that you want to delete, like we did above, you can avoid loops and filters and just delete it directly in a very clean and flat manner.

Deleting All Values

You can also easily delete all the values in a Set without looping by calling the clear method on it:

numbers.clear();

// Set []

You can see that a Set has a very clear and easy to use API versus an Array where you have to put in a bit more effort to delete a value.

Looping

Just like Arrays, you're often going to have to loop through a Set's values to work with them in some way and there's a couple of easy methods to do so and they both work the same on Sets and Arrays so you might not even have to change your current loops at all!

The first simple way to loop through the values of a Set is to use a for in loop:

for (const num of numbers) {

  // Num is now the value at the current loop position in the Set. 1, 2, 3, etc.
  console.log(num);

}

This is a very straightforward way to loop and if you aren't using this even with your Arrays you should be thinking about switching your loops to this. The benefit of a for in loop is that you don't have to use num[i] like you would in a regular for loop to get the value at the current index it's just the index by default. However, if your loop needs to keep track of the what position in the Array you are, then you should definitely stick to a regular for loop so you have that value readily available instead of having to keep track of it separately in some sort of count variable.

The second way to loop through the values of a Set is with a forEach loop. This functions pretty identical to a for in but it is accepted in more browsers (the difference isn't much these days they even both work in IE10 you should use caniuse if you need more details on this comparison).

numbers.forEach((num) => {

  // Just like above, num is the value in the current loop position in the Set.
  console.log(num);

});

Again this on the outside works the same as the for in loop as in you don't have access to the current index (which if you need to track a Set probably isn't best solution for you since position is not guaranteed).

Size

Last but not least, Sets come with a gettr, size, that returns the current amount of values in that Set:

const numOfNumbers = numbers.size;

// 6

This is pretty straight forward, just keep in mind that it is a property of the Set object and not a method.

It's Not All Unique

So at the beginning of this tutorial, I mentioned that all values in a Set are unique, as in there's no duplicate values, but that's not always the case with complex value types. Values that are objects can inserted multiple times in the Set because equality of Objects is a funny thing and you can see this below:

const superman = {
  weakness: "kyrptonite",
  occupation: "journalist"
};

const manThatIsSuper = {
  weakness: "kyrptonite",
  occupation: "journalist"
};

superman === manThatIsSuper; // false

You can see that although both objects share the same keys and values, they are not equal. This can be a tricky subject to pick up on and if you want a little more detail on the subject you can check out this tutorial but the basics of it is that they are both different instances of an object so they are not the same object and therefore they will both be inserted into the Set. If you want to check for unique objects you can easily find a deep equality function and use that to check equality before inserting it into the Set. Note that some tutorials might recommend extending the Set object to have deep object equality but I strongly recommend against modifying any prototypes you do not own as it can lead to problems down the road.

Conclusion

Hopefully by now you've been able to decide if any of your Arrays can be swapped over to Sets to take advantage of the unique nature of Set values and the easy to use API. If you're wondering why you should even bother using a Set even if you could benefit from it if your existing Array already works, just remember that it's never a bad thing to be more concise with your JavaScript and using a Set will let any other developer know that it is a specific kind of value collection and they can more easily grasp the purpose of your program. There is of course more a Set can do and if you're really interested you can browse over the MDN documentation on Sets for more in depth information.
